RESTService Resource

A restful web service that implements calls to the SPIN rule engine. It implements methods for rule inference and for constraints checking. Contains also a command line interface.

Simple test method

POST /runInferencesPost

This POST method call will execute the rule engine inference on the specified data. The required parameter is the base URI, that will be used to gather the rules and models.

Request Parameters
name type description
SPINURI form The URI containing the SPIN rules, if separate and not imported by the base model
SPINdocument form The actual SPIN rule document, in case it's not accessible or available at the specified URI
baseURI form The base URI for SPIN rule and models.
document form The actual document in case it's not accessible or available at the specified URI
outFormat form the result output format, default TTL. Options are TTL, NTRIPLES, RDFXML, N3, JSONLD, RDFJSON. For the complete list, see formats defined in the JENA API: https://jena.apache.org/documentation/io/rdf-output.html
Response Body
media type data type description
application/json SPINResults (JSON) a JSON result set in the form of a SPINResults format

Example

Request
POST /runInferencesPost
Accept: application/json
Response
HTTP/1.1 201 Created
Content-Type: application/json

  "newTriplets" : "...",
  "consoleLog" : "...",
  "jsonModel" : "...",
  "jsonNewTriplets" : "...",
  "jsonConstraints" : "..."
}

name type description default constraints
